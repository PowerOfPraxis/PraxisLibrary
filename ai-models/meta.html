About Meta AI

Meta AI™

The research organization behind the Llama family, the most widely adopted open-weight frontier model lineup in the industry.

Mission and Philosophy

Meta AI is the research division behind the Llama family of large language models. Unlike most frontier AI labs, Meta publishes Llama models with open weights under a permissive community license, letting researchers and developers download, modify, self-host, and fine-tune frontier-class models without the latency and cost of a hosted API.

That choice has reshaped the industry. Every major open-source AI tool built in the last two years, local inference runtimes, fine-tuning frameworks, quantization libraries, and privacy-first deployment stacks, assumes Llama-class weights as its baseline.

The Llama 4 Lineup

Llama 4 introduced a mixture-of-experts architecture across two primary variants, each tuned for a specific trade-off:

  • Llama 4 Maverick is the flagship, a 400B parameter MoE with 17B active parameters, tuned for the highest quality on the hardest workloads.
  • Llama 4 Scout is the efficient sibling, 109B parameters with 17B active and a 10M token context window, built for long-document and long-context workflows at lower infrastructure cost.
  • Llama 3.3 70B remains the workhorse dense model for production deployments that need predictable latency, lower VRAM footprint, and a simpler serving stack than a MoE model requires.

Open Weights and Deployment Posture

Every current Llama model ships with downloadable weights and a permissive community license, which means teams can run Llama on their own GPUs, in air-gapped environments, on cloud marketplaces, and in regulated industries where sending data to a third-party API is a compliance blocker.

Llama is available through Meta’s own distribution channels plus every major cloud provider, AWS Bedrock, Azure AI, Google Cloud Vertex AI, IBM watsonx, and Oracle OCI, and through Hugging Face for direct download. The same weights power all of them, so behavior and quality stay consistent regardless of where the model runs.

Feature Breakdown

Mixture of Experts
Native

400B total parameters with 17B active per token via MoE routing, delivering frontier-class quality at a fraction of the inference cost of equivalent dense models.

Vision and Image Input
Yes

Native multimodal image understanding, enabling chart analysis, document OCR, and visual reasoning across multiple images per request.

Tool Use and Function Calling
Yes

Structured function calling and tool use support, enabling agentic workflows that combine reasoning with external API calls and code execution.

Structured Output
Yes

JSON-mode and schema-adherent output for downstream parsing and integration with production pipelines.

Long-Context Reasoning
Yes

1M token context window lets Maverick handle entire books, multi-document corpora, and long codebases in a single request without chunking.

Self-Hosting
Native

Downloadable weights mean Maverick can run on your own GPU fleet, in air-gapped environments, or on the cloud marketplace of your choice.

Fine-Tuning
Yes

Full fine-tuning, LoRA, and QLoRA supported via the open-source ecosystem (PEFT, Axolotl, Unsloth), unlocking domain adaptation that hosted APIs typically cannot offer.

Quantization
Yes

Community-maintained quantized variants (GGUF, AWQ, GPTQ) make Maverick viable on consumer and prosumer hardware, not just data center GPUs.

Data Handling and Governance

Training on Customer Data

Not applicable. You run Maverick on your own infrastructure, so inputs and outputs never leave your environment unless you choose to route them through a cloud provider.

Data Retention

Fully controlled by the deploying team. Self-hosted deployments give complete data sovereignty.

Compliance Fit

Ideal for HIPAA, GDPR, SOC 2, and air-gapped environments where third-party API calls are prohibited. Compliance inherits from your own infrastructure posture.

Geographic Availability

Anywhere you can run the weights. No regional API limitations, no data residency surprises.

License

Llama Community License, permissive for commercial use with a few scale-based conditions for the largest deployments.

Inference Infrastructure

Requires data center GPU capacity for the full model, or quantized variants for smaller footprints. MoE routing means inference cost is closer to a 17B dense model than a 400B one.

Best Use Cases

Air-Gapped and Regulated Deployments

Healthcare, defense, legal, and financial workloads where data cannot leave the organization’s perimeter. Self-hosted Maverick runs entirely on your infrastructure.

Long-Document Enterprise Analysis

1M token context plus strong reasoning handles whole contracts, clinical records, and multi-report research synthesis in a single call.

Domain-Specific Fine-Tuning

Fine-tune Maverick on proprietary legal, medical, financial, or engineering corpora to build a custom model that hosted APIs cannot match on your specific domain.

Cost-Predictable Production

Fixed infrastructure cost per month rather than per-token API billing, valuable for high-volume workloads where API costs would be prohibitive.

Research and Experimentation

Open weights unlock interpretability research, mechanistic analysis, and novel fine-tuning techniques that closed models make impossible.

What Llama 4 Maverick Excels At

  • Data sovereignty. Maverick is the only tier of frontier-class AI quality that can run entirely on your own infrastructure, which is a hard requirement for regulated industries and a significant advantage everywhere else.

  • Customization depth. Open weights enable fine-tuning, LoRA adaptation, and quantization workflows that closed models cannot offer, letting teams build genuinely specialized systems.

  • Long-term cost predictability. Self-hosted inference is a fixed infrastructure line item rather than a variable per-token API bill, which matters enormously at production scale.

Best Use Cases

Whole-Codebase Engineering

Long Video and Transcript Analysis

Hours of meeting transcripts, lecture recordings, and interview content fit in a single call without chunking, enabling faithful full-document synthesis.

Entire Document Library Q&A

Load a complete policy library or research corpus into a single request for grounded answers that cite across the full source material.

Lower-Cost Self-Hosted Production

Smaller parameter count than Maverick means lower GPU requirements and faster inference, ideal for teams that need open-weight flexibility at production scale.

What Llama 4 Scout Excels At

  • Unmatched context length. The 10M token window is the largest in any open-weight model, unlocking whole-corpus workflows that no hosted API currently offers at this context size.

  • Infrastructure accessibility. Smaller parameter footprint than Maverick makes Scout practical for teams without data center GPU capacity, democratizing long-context work.

  • Open-weight flexibility. Everything that Maverick offers in terms of fine-tuning, quantization, and self-hosting applies to Scout too, at lower cost and complexity.

  • Hardware accessibility. Quantized 70B runs on a single high-end GPU or Apple Silicon workstation, democratizing frontier-class AI to individual developers and small teams.
