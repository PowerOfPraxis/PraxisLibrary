Self-Verification

Validate answers through systematic checking.

The Concept

Trust But Verify

Self-Verification adds a validation step after answer generation. Rather than accepting the model's first response as final, this approach asks the model to check whether its answer actually satisfies the original requirements.

Core Insight

Verification is often easier than generation. A model that produces an incorrect answer can frequently identify the error when explicitly asked to check its work.

The Pattern

1) Generate answer. 2) Formulate verification checks. 3) Execute checks. 4) Accept or regenerate.

Strategies

Verification Methods

Backward Verification

Work backwards from the answer to see if it leads to the original problem conditions. Powerful for math and logic.

Constraint Checking

Verify that the answer satisfies all stated and implied constraints from the original problem.

Sanity Checks

Apply common-sense validation to catch obviously wrong answers, even without formal verification.

Example

Verification in Action

Problem & Answer

Problem: "If 3x + 7 = 22, what is x?"
Answer: x = 5

Backward Verification

Check: Substitute x = 5 back into equation
3(5) + 7 = 15 + 7 = 22 ✓

Result: Matches original equation. Answer verified.

Problem

"Schedule a 2-hour meeting for 5 people next week, avoiding Monday mornings and Friday afternoons."

Verification Checklist

✓ Is the duration exactly 2 hours?
✓ Can all 5 people attend?
✓ Is it scheduled for next week?
✓ Does it avoid Monday morning?
✓ Does it avoid Friday afternoon?

Implementation

Verification Patterns

Single-Turn

Include verification instructions in the original prompt, asking the model to check before finalizing.

Two-Turn

Generate answer first, then send separate verification prompt with the answer and original problem.

With Regeneration

If verification fails, regenerate with feedback about what went wrong. Repeat until verified.
