Example Selection

KNN Prompting

Select the most relevant examples using nearest-neighbor retrieval.

The Concept

Dynamic Example Selection

KNN Prompting retrieves few-shot examples that are most similar to the current input. Instead of using fixed examples, it dynamically selects from a pool based on semantic similarity, ensuring the demonstrations are maximally relevant.

How It Works

Encode examples and the query into embeddings, find the k closest examples to the query, and use those as few-shot demonstrations.

Requirements

1) Pool of labeled examples. 2) Embedding model for similarity. 3) Retrieval mechanism to find nearest neighbors.

Process

KNN Pipeline

1. Embed

Convert all pool examples and the input query into vector embeddings.

2. Retrieve

Find the k examples with highest similarity to the query embedding.

3. Prompt

Use retrieved examples as few-shot demonstrations in the prompt.

Benefits

Why Use KNN

Relevance

Similar examples provide better guidance than random or generic ones.

Scalability

Can leverage large example pools without fitting all into the prompt.

Adaptability

Automatically adjusts to different input types and domains.
