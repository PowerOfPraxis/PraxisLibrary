Reflexion

Learn from failure through reflection and memory.

The Concept

Reflective Learning

Reflexion enables agents to learn from their mistakes without updating model weights. When a task fails, the agent generates a verbal reflection analyzing what went wrong. These reflections are stored in memory and used to guide future attempts at similar tasks.

Key Innovation

Unlike simple retry mechanisms, Reflexion creates persistent lessons learned. The agent doesn't just try againâ€”it tries differently based on explicit self-analysis.

The Cycle

1) Attempt task. 2) Receive feedback. 3) Reflect on failure. 4) Store reflection. 5) Retry with memory context.

Components

Reflexion Architecture

Actor

The agent that attempts the task. Takes actions based on current state and memory context.

Evaluator

Assesses whether the attempt succeeded or failed, providing feedback signals.

Self-Reflection

Generates verbal analysis of failures and concrete suggestions for improvement.

Example

Reflexion in Action

Attempt 1 - Failed

Task: Write function to check palindrome
Code: return s == s[::-1]
Tests fail on "A man, a plan, a canal: Panama"

Reflection & Retry

Reflection: "My solution failed because I didn't handle case sensitivity or non-alphanumeric characters. Palindrome checks should normalize the string first."

Memory: Store this insight

Attempt 2: Normalize string, filter characters, then compare. Tests pass.
