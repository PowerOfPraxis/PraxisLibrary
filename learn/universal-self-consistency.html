Universal Self-Consistency

Extend consistency voting to open-ended generation.

The Concept

Beyond Fixed Answers

Universal Self-Consistency (USC) extends self-consistency from tasks with discrete answers to free-form generation. Instead of simple voting, it uses the LLM itself to identify the most consistent response among multiple samples.

The Challenge

Standard self-consistency needs identical answers to vote. USC handles cases where responses differ in wording but agree in meaning.

Process

1) Generate multiple responses. 2) Ask LLM to identify which response is most consistent with the others. 3) Select that response.

Applications

When to Use USC

Open QA

Questions with free-form answers that can be phrased multiple ways.

Summarization

Finding the summary that best captures consensus across multiple attempts.

Code Generation

Selecting the implementation most consistent with multiple solution attempts.
