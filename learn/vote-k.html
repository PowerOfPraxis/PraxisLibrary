Active Learning

Vote-K

Let the model vote on which examples to annotate.

The Concept

Selective Annotation

Vote-K is an active learning approach where the model helps select which unlabeled examples would be most valuable to annotate. By voting on candidate examples, it identifies the ones most likely to improve performance when added to the demonstration pool.

Key Benefit

Reduces annotation effort by focusing labeling work on examples the model finds most informative, rather than randomly selected ones.

The Process

1) Present unlabeled candidates. 2) Model votes on usefulness. 3) Annotate top-voted examples. 4) Add to demonstration pool.

Workflow

Vote-K Steps

Candidate Pool

Gather unlabeled examples that could potentially be annotated and used as demonstrations.

Voting

Ask the model to select k examples it believes would be most helpful if labeled.

Integration

After annotation, add voted examples to improve future few-shot prompts.

Use Cases

When Vote-K Helps

Limited Budget

When you can only afford to label a small number of examples.

Iterative Building

Progressively growing a demonstration set over multiple rounds.

Domain Coverage

Ensuring examples cover diverse aspects of the problem space.
