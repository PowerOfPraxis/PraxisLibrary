Recursion of Thought

Recursive problem decomposition with nested reasoning.

The Concept

Recursive Decomposition

Recursion of Thought applies the divide-and-conquer strategy recursively. When a sub-problem is still too complex, it spawns its own decomposition, creating a tree of reasoning chains that bottoms out at solvable base cases.

Like Programming Recursion

Just as recursive functions call themselves on smaller inputs, RoT applies the same decomposition strategy at each level until reaching simple base cases.

Pattern

1) Check if base case. 2) If not, decompose. 3) Recursively solve each part. 4) Combine results.

Applications

When to Use RoT

Hierarchical Problems

Tasks with natural hierarchical structure that mirrors recursive decomposition.

Variable Depth

Problems where complexity varies and fixed decomposition levels won't work.

Self-Similar Structure

Sub-problems that have the same form as the original problem.
